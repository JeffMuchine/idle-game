# DESIGN.md — Dungeon Eternal

Generated by /plan-design-review on 2026-03-26. Source of truth for all visual decisions.

---

## Design Identity

**Game:** Dungeon Eternal — D&D idle browser game
**Aesthetic:** Dark Gothic Fantasy — think Baldur's Gate 3 UI, not Candy Crush
**Feeling:** Power, weight, mystery. The interface should feel like parchment and iron, not glass and blur.
**Anti-pattern:** No purple-gradient generic SaaS. No rounded bubbly cards. No icons in colored circles. No centered layouts.

**Two typefaces only:**

## Interaction States — Specifics

### First Time Player
- Hero creation modal on first load: choose class (4 cards, not icons in circles — use large visual with class art description)
- No empty inventory message on first load: just the 6 empty equipment slots with tooltips
- Combat starts automatically after class selection — NO confirmation needed

### Returning Player (load)
- If offline > 5 min: show offline progress modal BEFORE resuming combat
- Modal: dark overlay, centered card, Cinzel heading "Welcome Back, [Name]"
- Shows: time away, gold earned, kills (approximated), any notable finds
- One action: "Continue Adventure" button

### Prestige / Ascension
- Button only appears when player has reached floor 50+
- "ASCEND" button in bottom bar: gold color, slightly pulsing when available
- Click: full-screen overlay — dark fade, then dramatic reveal
  - Cinzel heading: "You have Ascended"
  - New tier badge animates in
  - AP earned shown with counter animation
  - "Claim Rewards" CTA — gold button

## Accessibility

- **Color contrast:** All text/background combos must pass WCAG AA (4.5:1 for body, 3:1 for large text)
  - `--text-primary` (#e8dcc8) on `--bg-panel` (#13131a): 11.4:1 ✓
  - `--gold-primary` (#c9a84c) on `--bg-panel` (#13131a): 5.1:1 ✓
  - `--rarity-common` (#9d9d9d) on `--bg-panel`: 5.2:1 ✓
- **Touch targets:** Minimum 44×44px for all interactive elements on mobile
- **Keyboard navigation:** Tab through panels; equipment slots focusable; modal traps focus
- **ARIA:** `role="progressbar"` on HP/XP bars with `aria-valuenow`. `role="log"` on combat log. `aria-live="polite"` on toast container.
- **Reduced motion:** `@media (prefers-reduced-motion: reduce)` — disable all keyframe animations, keep transitions ≤ 100ms

## AI Slop Prevention Checklist

- [ ] No purple/violet/indigo gradient backgrounds (use near-black + gold)
- [ ] No 3-column icon-in-circle feature grid anywhere
- [ ] No centered text-align on all headings
- [ ] No uniform large border-radius (we use 2-6px max)
- [ ] No decorative blobs or wavy SVG dividers
- [ ] No emoji as bullet points or decorative elements
- [ ] No colored left-border on cards
- [ ] No "Welcome to Dungeon Eternal" hero copy
- [ ] No cookie-cutter section rhythm (this is a game, not a landing page)

## Decisions Log

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Font pairing | Cinzel (display) + Crimson Pro (body) | Cinzel has the Roman imperial weight that reads as D&D authority; Crimson Pro is legible at 11-13px UI sizes where sans-serifs look clinical |
| Background palette | Near-black (#0d0d0f) not pure black | Pure black looks like a dev console; near-black with slight blue tint (#111118) reads as "dungeon atmosphere" |
| Accent color | Gold (#c9a84c) not bright yellow | Bright yellow reads modern/tech; desaturated gold reads medieval/earned; bright gold (#ffd700) reserved for legendary only |
| Border radius | 2-6px hard cap | Rounded corners soften; D&D fantasy should feel angular, forged, not soft. No rounded-xl anywhere. |
| Rarity colors | WoW standard (gray/green/blue/purple/orange) | Players already know this language; inventing new colors costs recognition. Don't fight muscle memory. |
| Combat log | 3 lines max, font-xs monospace-adjacent | More lines = noise; fewer = missing feedback. 3 is the magic number for combat games (Path of Exile, Diablo). |
| Mobile layout | Bottom tab bar (not sidebar drawer) | Idle games are played one-handed; thumb reach to bottom tab bar is natural; sidebar drawer requires two-handed interaction |
| Toasts | Achievement: bottom-right slide-in / Legendary: center-screen | Achievement is frequent (low-key delight); legendary is rare (stop-what-you're-doing moment). Different gravity for different weight. |
| Number formatting | Compact: 1K/1M/1B/1T, then scientific (1.2e15) | Players understand K/M/B; scientific is a last resort for overflow. Never raw 7-digit integers in UI. |
| Equipment slot count | 6 slots (weapon/armor/helmet/ring/amulet/offhand) | D&D canonical; enough to feel like a full loadout without overwhelming. |
| Prestige animation | Full-screen wipe + fanfare | First prestige is the emotional peak of the game. It must feel like an event, not a button click. |
